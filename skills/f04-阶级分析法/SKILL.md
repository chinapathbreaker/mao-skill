# 阶级分析法

## R — 原文 (Reading)

> "谁是我们的敌人？谁是我们的朋友？这个问题是革命的首要问题。中国过去一切革命斗争成效甚少，其基本原因就是因为不能团结真正的朋友，以攻击真正的敌人。"
>
> — 毛泽东，《中国社会各阶级的分析》，1925年12月

---

## I — 方法论骨架 (Interpretation)

阶级分析法是"敌友判断"的核心工具，核心是**不以感情、意识形态判断敌友，以利益关系判断敌友**。

阶级光谱五层：
1. **铁杆支持者**：利益完全绑定，死忠，不离不弃
2. **可争取者**：利益相关，但立场不坚定，可以被影响
3. **摇摆者**：利益不直接相关，随大流
4. **可瓦解者**：利益在对手阵营，但内部有裂缝
5. **坚定反对者**：利益完全对立，不可调和

关键心法：**真正的失败不是因为不够努力，而是因为敌友判断错误——把朋友当敌人，把敌人当朋友。**

## A1 — 书中的应用 (Past Application)

### 案例 1：中国社会各阶级的分析（1925）

- **问题**：国共合作初期，各种政治力量都想拉拢革命阵营，但不知道该团结谁、反对谁。
- **方法论的使用**：用经济剥削关系作为判断敌友的标准，构建阶级光谱——地主买办（敌人）vs 民族资产阶级（可争取）vs 小资产阶级（朋友）vs 工人农民（铁杆支持）。
- **结论**：革命的首要问题是搞清楚谁是我们的敌人。
- **结果**：这个阶级光谱成为整个革命时期的指导框架。

### 案例 2：湖南农民运动考察（1927）

- **问题**：农民运动迅猛发展，党内对农民运动有两种极端判断。
- **方法论的使用**：对农民进行阶级分层——雇农（铁杆）、贫农（主要力量）、中农（团结）、富农（动摇）、地主（敌人）。
- **结论**：革命者应该支持雇农和贫农，这是革命的主要力量来源。
- **结果**：为土地革命路线奠定基础。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill？

1. 判断一件事中，谁是敌人、谁是朋友
2. 构建联盟，不知道该拉拢谁
3. 面对多方博弈，不知道谁会帮你、谁会坑你
4. 分析竞争对手的团队/公司，不知道谁是核心人物
5. 推动变革，不知道阻力来自哪里

### 语言信号

- "这件事谁会支持谁会反对"
- "不知道该拉拢谁"
- "多方博弈怎么破"
- "这个团队里谁是真正的决策者"
- "这个人到底站哪边"
- "我的盟友在哪里"

### 与相邻 skill 的区分

- 与 `maoxuan-enemy-friend-dynamics` 的区别：阶级分析法是静态的"利益关系判断"，敌友判断动态框架是"利益关系会如何变化"
- 与 `maoxuan-contradiction-analysis` 的区别：阶级分析法用于"人"，矛盾分析法用于"事"

## E — 可执行步骤 (Execution)

当 skill 被激活后，按以下步骤执行：

⚠️ **确认触发**：在开始之前，确认此场景适合使用本skill（输入'继续'进入，输入'取消'退出）

1. **列出所有参与者**
   - 完成标准：写下这件事中涉及的所有利益方（不只是"人"，也是"组织/群体"）

⏸️ **检查点**：请确认这个参与者列表是否完整（输入'继续'或提出修正）

2. **分析每个参与者的利益基础**
   - 完成标准：对每个参与者回答——利益来源是什么？依赖谁？和我的利益关系？（一致/冲突/无关）

⏸️ **检查点**：请确认各参与者的利益关系判断是否准确（输入'继续'或提出修正）

3. **绘制阶级光谱**
   - 完成标准：将所有参与者放在五层光谱上（铁杆→可争取→摇摆→可瓦解→坚定反对）

⏸️ **检查点**：请确认阶级光谱的划分是否符合你的判断（输入'继续'或提出修正）

4. **制定差异化策略**
   - 完成标准：每类参与者都有明确策略（依靠/争取/观察/瓦解/减少破坏）

5. **定期更新**
   - 完成标准：写下"什么情况下阶级光谱会发生变化"

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **单一对手**：只需要判断打不打，不需要画光谱
- **纯技术问题**：技术评估不需要阶级分析
- **信任完全缺失**：在没有信任基础的关系中，利益分析可能失效

### 作者在书中警告的失败模式

- 以意识形态而非利益判断敌友（"我喜欢谁就拉拢谁"）
- 试图争取所有人（"谁都不敢得罪"）
- 把敌人绝对化（"这个人永远不可能站到我这边"）
- 静态看待敌友关系
