# 持久战三阶段框架

## R — 原文 (Reading)

> "第一个阶段，是敌之战略进攻、我之战略防御的时期。第二个阶段，是敌之战略保守、我之准备反攻的时期。第三个阶段，是我之战略反攻、敌之战略退却的时期。"
>
> — 毛泽东，《论持久战》，1938年5月

---

## I — 方法论骨架 (Interpretation)

任何长期竞争格局（战争、商业竞争、技术追赶）都可以用**三阶段模型**分析。

三阶段：
1. **防御阶段**：对手强，我弱。对手在进攻，我在战略收缩。
2. **相持阶段**：双方力量接近，拉锯。这是"最困难"的阶段，多数人在这里放弃。
3. **反攻阶段**：我方力量超过对方，发起战略进攻。

关键洞察：**相持阶段是最容易被误判的阶段。**
- 速胜论者在防御阶段就想要胜利，被打败后变成投降论
- 投降论者在相持阶段就放弃，看不到黎明前的黑暗
- 真正的高手知道：**相持阶段越长，积累的优势越大，反攻时越彻底**

## A1 — 书中的应用 (Past Application)

### 案例 1：中日战争的三个阶段（1937-1945）

- **问题**：1937年抗日战争爆发，中国极弱日本极强。能赢吗？多久能赢？
- **方法论的使用**：用三阶段分析——防御（1937-1938）、相持（1938-1944）、反攻（1944-1945）。
- **结论**：不是速胜，不是亡国，是持久战，关键是在相持阶段积累力量。
- **结果**：1938年的预判与1945年的实际结果高度吻合。

### 案例 2：苹果反攻PC市场的三个阶段（1976-至今）

- **问题**：1985-2001年苹果在PC市场节节败退，还有希望吗？
- **方法论的使用**：三阶段分析——防御（1976-1985）→ 相持（1985-2001）→ 反攻（2001-至今，iPod→iPhone→M芯片）。
- **结论**：乔布斯回归是转折信号，相持阶段越长积累越深。
- **结果**：苹果成为全球市值最高公司，这不是一蹴而就，是数十年积累。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill？

1. 判断一个长期趋势（某行业、某技术、某个人的发展）
2. 处于弱势，但判断不了还有没有希望
3. 处于强势，但判断不了什么时候可以发起总攻
4. 被短期失败打击，怀疑方向是否正确
5. 想要速胜，但发现对手比自己强太多

## E — 可执行步骤 (Execution)

当 skill 被激活后，按以下步骤执行：

1. **判断当前处于哪个阶段**
   - 完成标准：能明确回答"防御/相持/反攻"，并说明判断依据

2. **针对当前阶段采取正确策略**
   - 完成标准：知道该阶段的核心策略和常见错误

3. **识别转折信号**
   - 完成标准：说出"什么信号出现时，意味着相持→反攻的转折点到了"

4. **双轨运行（战略持久+战术速决）**
   - 完成标准：能区分"战略层面"和"战术层面"的不同要求

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **短期决策**：持久战是战略框架，不适用于需要快速决断的短期问题
- **力量悬殊的碾压局**：当一方有压倒性优势且不会变化时，不需要持久战框架
- **纯信息查询**：只需要了解现状，不需要判断阶段

### 作者在书中警告的失败模式

- 速胜论（在防御阶段想要胜利，资源耗尽）
- 投降论（在相持阶段放弃，黎明前的黑暗里缴械）
- 反攻阶段保守（力量对比逆转但不敢决战）
