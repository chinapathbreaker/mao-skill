# 又团结又斗争的动态平衡框架

## R — 原文 (Reading)

> "以斗争求团结则团结存，以退让求团结则团结亡。"
>
> — 毛泽东，《目前抗日统一战线中的策略问题》，1940年3月

---

## I — 方法论骨架 (Interpretation)

又团结又斗争是处理**盟友关系分歧**的核心原则，核心是动态平衡。

两个原则：
- **团结为目的**：为了扩大力量基础，但团结不能没有底线
- **斗争为手段**：为了防止同盟者偏离方向，但斗争不能破坏团结基础

四种错误：
1. **只有团结没有斗争**→ 尾巴主义（失去原则）
2. **只有斗争没有团结**→ 关门主义（缩小阵营）
3. **斗争过早过猛**→ 破坏团结基础
4. **斗争过晚过轻**→ 错失纠正时机

---

## A1 — 书中的应用 (Past Application)

### 案例 1：皖南事变（1941）的处理

- **问题**：1941年，国民党发动皖南事变，突袭新四军。这是统一战线内部最严重的冲突——要不要彻底破裂？
- **方法论的使用**：中共判断——皖南事变是国民党顽固派的挑衅，但主要敌人（日本）还在，民族矛盾仍然是主要矛盾，不能因为皖南事变就彻底破裂统一战线。因此：政治上强烈抗议（斗争），军事上重组新四军继续抗日（团结）。没有全面破裂，但也没有忍气吞声。
- **结论**：斗争是为了更好地团结，不是为了破裂。
- **结果**：统一战线维持到了抗战胜利。

### 案例 2：抗日统一战线的日常维护（1937-1945）

- **问题**：统一战线中，国民党经常搞摩擦（皖南事变只是最严重的一次）。怎么处理这些摩擦？
- **方法论的使用**：斗争——对国民党的摩擦在政治上要揭露和抗议，不能忍气吞声；团结——同时坚持抗日大方向不变，不把摩擦升级为全面破裂。
- **结论**：又团结又斗争的动态平衡。
- **结果**：统一战线维持8年，保证了抗战的顺利进行。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要 this skill？

1. 联盟/合作中出现分歧，不知道怎么处理
2. 盟友提出了不合理的要求，不知道该拒绝还是妥协
3. 团队内部有不同意见，不知道该说还是不该说
4. 想要扩大阵营，但又不想失去原则
5. 面临"尾巴主义"还是"关门主义"的两难

### 语言信号

- "盟友和我有分歧怎么办"
- "要不要斗争"
- "这个关系怎么处理"
- "团结为主还是斗争为主"
- "能不能撕破脸"
- "这是不是尾巴主义"

### 与相邻 skill 的区分

- 与 `maoxuan-unity-front` 的区别：统一战线策略是整体框架，又团结又斗争是统一战线的核心操作原则
- 与 `maoxuan-rational-advantage-limit` 的区别：又团结又斗争是"团结/斗争"的平衡，有理有利有节是"让步/收获"的平衡

## E — 可执行步骤 (Execution)

当 skill 被激活后，按以下步骤执行：

1. **判断团结的目标还在吗**
   - 完成标准：回答"核心利益是否仍然一致，破裂的代价有多大"

2. **判断斗争的边界在哪里**
   - 完成标准：能说明"斗争的底线在哪里（不能把朋友变成敌人）"

3. **判断团结和斗争的比例**
   - 完成标准：根据情境，决定"当前是以团结为主还是以斗争为主"

4. **定期评估平衡**
   - 完成标准：能回答"这个平衡需要调整的信号是什么"

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **敌我矛盾**：这是人民内部矛盾的处理方式，不适用于敌我矛盾（需要对抗而非动态平衡）
- **一次性博弈**：又团结又斗争需要多次互动才有意义，一次性交易不适用
- **道德底线被触碰**：团结有底线，基本伦理不能让步

### 作者在书中警告的失败模式

- 尾巴主义（只有团结没有斗争，失去原则）
- 关门主义（只有斗争没有团结，缩小阵营）
- 斗争过早过猛（破坏团结基础）
- 斗争过晚过轻（错失纠正时机）
