# 群众路线决策法

## R — 原文 (Reading)

> "从群众中集中起来到群众中坚持下去，以形成正确的领导意见，这是基本的领导方法。"
>
> — 毛泽东，《关于领导方法的若干问题》，1943年6月

---

## I — 方法论骨架 (Interpretation)

群众路线是毛选方法论中**最核心的决策框架**，不是"民主决策"的道德口号，而是一套**可操作的决策循环**。

三阶段循环：
1. **从群众中来**：深入群众，收集分散的、无系统的意见（不是只听同意你的人）
2. **领导研究**：集中讨论，化为系统的决策方案（补充专业分析）
3. **到群众中去**：向群众解释决策（不是发文件，是讲清楚为什么），用实践检验

关键区分：
- 不是"先调查后决策"的两步法
- 是"调查→决策→执行→再调查→再决策"的持续循环
- 每个循环都比上一次更接近正确

## A1 — 书中的应用 (Past Application)

### 案例 1：延安整风运动（1942-1945）

- **问题**：党内存在严重的教条主义倾向，理论脱离实际，如何纠正？
- **方法论的使用**：整风运动不是自上而下发文件批判，而是让全党一起学文件、一起反省——每个党员用实事求是核查表自查→集体讨论形成新的认识→再到实践中验证。
- **结论**：解决思想问题需要群众的广泛参与，不是靠权威压服。
- **结果**：整风运动统一了全党思想，形成了解放战争时期的强大组织凝聚力。

### 案例 2：土地改革中的群众路线（1946-1949）

- **问题**：解放战争时期，如何动员农民支持战争？
- **方法论的使用**：不是政府发文件分配土地，而是让农民自己起来斗争——先调查了解农民真正需要什么→制定政策框架→让农民执行，在执行中调整。
- **结论**：群众是真正的铜墙铁壁，赢得群众支持比武器装备更重要。
- **结果**：土地改革为数百万解放军提供了兵源和物资支持。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill？

1. 做重要决策，不知道怎么平衡"专业判断"和"群众意见"
2. 政策/产品推不下去，执行层不认同
3. 想要变革，但阻力太大
4. 领导一个团队，不知道怎么让团队真正认同
5. 觉得自己的想法"正确"但得不到支持

### 语言信号

- "这个决策怎么做"
- "群众意见怎么收集"
- "推不动"
- "为什么执行层不认同"
- "我的方案是对的但没人听"
- "怎么让团队认同"

### 与相邻 skill 的区分

- 与 `maoxuan-investigation-research` 的区别：调查研究是收集信息，群众路线是决策循环（信息→决策→执行→反馈）
- 与 `maoxuan-introspection-criticism` 的区别：整风反思是组织层面的检查，群众路线是决策层的循环框架

## E — 可执行步骤 (Execution)

当 skill 被激活后，按以下步骤执行：

1. **从群众中来**
   - 完成标准：收集了至少3个不同立场的人的意见（不能只听同意你的人）

2. **领导研究**
   - 完成标准：把分散意见化为系统的决策方案，并补充专业分析

3. **到群众中去**
   - 完成标准：用通俗的语言向执行者解释"为什么这个决策是对的"

4. **收集反馈，回到第1步**
   - 完成标准：能识别"决策执行后的反馈中，有哪些信号说明需要调整"

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **紧急危机**：来不及走群众路线，但危机后要复盘
- **单一专家领域**：需要高度专业化的决策（如航天工程）
- **道德底线**：涉及基本伦理的决策不能靠投票决定

### 作者在书中警告的失败模式

- 只听一方面人的话（群众路线变成了"听我的话"）
- 调查完束之高阁（没有真正把群众意见纳入决策）
- 把群众路线当成道德口号（没有可操作步骤）
